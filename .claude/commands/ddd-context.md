ウィトゲンシュタインの「言語ゲーム」の観点から、ドメインの境界付けられたコンテキストを分析してください。

言語ゲームとは「言葉の意味は、その言葉が使われる文脈（目的・状況・参加者）によって決まる」という哲学概念です。DDDの境界付けられたコンテキストとユビキタス言語はこの概念で説明できます。

## 分析手順

### Step 1: アクターと目的の洗い出し

このドメインに関わるアクター（人・外部システム）を列挙し、それぞれの目的を明確にしてください。

```
アクター → 目的 → そのアクターが使う言葉とその意味
```

### Step 2: 言語ゲームの発見

同じ用語が異なるアクター・目的の間で異なる意味を持つケースを探してください。

例:「商品」という言葉でも:
- 商品審査スタッフにとっては「審査対象の情報セット」
- 配送業者にとっては「配送する物理的な荷物」
- 在庫スタッフにとっては「棚に並ぶ在庫単位」

この意味の分岐こそが境界付けられたコンテキストの境界です。

### Step 3: コンテキスト境界の確定

以下の4つの観点で境界を検証してください:

1. **プロセスサイクルの相違**: ライフサイクルや状態遷移が異なるか
2. **達成目標の違い**: 大粒度の目的が異なるか
3. **関連概念の相違**: 中心となる概念が大幅に異なるか
4. **不変条件の違い**: 守るべきルール・制約が異なるか

### Step 4: コンテキストマップの作成

各コンテキストを以下の形式で定義してください:

```
【コンテキスト名】
アクター: 誰が
目的: 何のために
ユビキタス言語: このコンテキスト内での用語定義
ドメインルール: このコンテキスト固有の業務ルール
不変条件: 常に満たすべき制約
```

コンテキスト間の関係（上流/下流、共有カーネル、腐敗防止層等）も明示してください。

### Step 5: DRY原則の誤用チェック

「名前が同じ」「構造が似ている」だけで安易に共通化されていないか検証してください。目的が異なるなら、たとえ構造が似ていても分離すべきです。唯一無二の真のモデルは存在しません。
