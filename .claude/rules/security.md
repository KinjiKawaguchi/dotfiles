# セキュリティ

## シークレット管理

- APIキー、パスワード、トークンをソースコードにハードコードしない。
- 環境変数またはシークレットマネージャーを使用する。
- `.env` ファイルは `.gitignore` に必ず追加する。

## 入力バリデーション

- ユーザー入力はシステム境界で必ずバリデーションする（Zod, Pydantic等）。
- SQLはパラメータ化クエリを使用し、文字列結合でクエリを組み立てない。
- HTMLの出力はサニタイズし、XSSを防止する。

## 認証・認可

- 全エンドポイントで認証・認可を検証する。
- CORSを適切に設定する。
- レートリミットを適用する。

## コミット前チェックリスト

1. ハードコードされたシークレットがないこと
2. ユーザー入力がバリデーションされていること
3. SQLインジェクション対策がされていること
4. XSS対策がされていること
5. CSRF対策がされていること
6. エラーメッセージに機密情報が含まれていないこと
